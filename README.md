# 股票量化选股系统

基于多因子策略的股票筛选与回测系统，支持阶梯放量、底部暴力K、异动检测等多种技术分析策略。

## 项目结构

```
wbzq/
├── main_par2.py          # 主程序：选股策略核心
├── batch_validate.py     # 批量验证工具
├── config.py             # 策略配置中心
├── data_manager.py       # 数据管理模块
├── validate_list.csv     # 验证股票列表
├── logs/                 # 验证日志目录
├── result.csv            # 验证结果汇总
├── result_detail.csv     # 验证结果详情
├── validation_report.txt # 验证报告
└── out.csv               # 选股结果输出
```

## 配置说明

编辑 `config.py` 文件配置策略参数：

```python
# API配置
TUSHARE_TOKEN = "your_token_here"  # Tushare API Token

# 策略阈值（可根据需要调整）
COLLECT_PCT_00_60 = 3.8      # 主板异动涨幅阈值
COLLECT_PCT_OTHER = 7.0      # 创业板/科创板异动涨幅阈值
VOLUME_SURGE_RATIO = 3.0     # 放量检测倍数
BODY_RATIO_THRESHOLD = 0.2   # 十字星判定阈值
```

## 使用方式

### 1. 单只股票调试

```bash
python main_par2.py --date 20250620 --days 60 --debug 688321.SH
```

参数说明：
- `--date`: 回测日期，格式YYYYMMDD，默认今天
- `--days`: 回测历史天数，默认60天
- `--debug`: 调试模式，传入股票代码查看详细检查过程

### 2. 批量选股

```bash
python main_par2.py --date 20250620 --days 60
```

### 3. 批量验证

编辑 `validate_list.csv` 添加验证股票：

```csv
code,name,date_str,days,expect_strategy
688321.SH,微芯生物,20250620,60,策略1
688799.SH,华纳药厂,20250508,60,策略1
600601.SH,方正科技,20250723,60,策略1
```

运行验证：

```bash
python batch_validate.py
```

验证输出：
- `logs/股票代码_日期.log` - 单只股票详细日志
- `result.csv` - 验证结果汇总
- `result_detail.csv` - 详细检查项结果
- `validation_report.txt` - 完整验证报告

## 选股策略逻辑

### 策略条件（11项检查）

| 序号 | 检查项 | 说明 |
|------|--------|------|
| 1 | 基础技术指标 | MACD>0，收盘价>MA60，MA60向上 |
| 2 | K线形态 | 小阳线/十字星/带长下影线阴线 |
| 3 | 振幅 | 主板≤4%，创业板/科创板≤7% |
| 4 | 阶梯放量 | J<13且满足阶梯放量条件 |
| 5 | 放量 | 周期内曾出现成交额≥前5天均值3倍 |
| 6 | 异动 | 周期内曾出现异动信号 |
| 7 | 成交额排名 | 当日成交额在前60% |
| 8 | 底部暴力K | 周期内有底部暴力K信号 |
| 9 | 派发信号 | 无主力出货信号（3种检测） |
| 10 | 知行多空线 | 中期多空线>多空线，收盘价≥多空线 |
| 11 | 次新股 | 非次新股（上市≥180天） |

### 主力出货信号检测

| 版本 | 逻辑 |
|------|------|
| V1 | 最高点当天放天量大阴线（成交额≥前日2倍） |
| V2 | 最高点后两天成交额均值放大且累计跌幅达标 |
| V3 | 最高点后出现2次及以上放量长阴 |

### 异动检测

| 类型 | 条件 |
|------|------|
| 收集区异动 | 上穿60日线后，放量上涨（主板≥3.8%，其他≥7%） |
| 堆量建仓 | 上穿60日线后，≥2次放量上涨 |
| 突破放量 | 突破60日线时放量≥2倍且涨幅≥3% |

## 常见问题

### Q: 如何调整策略阈值？
A: 编辑 `config.py` 中的 `StrategyThresholds` 类，修改对应参数后重新运行。

### Q: 验证失败如何排查？
A: 查看 `logs/股票代码_日期.log` 文件，找到标记为 `❌` 的检查项。

### Q: 如何添加新的验证股票？
A: 在 `validate_list.csv` 中添加一行，格式：`code,name,date_str,days,expect_strategy`

### Q: 数据从哪来？
A: 通过 Tushare API 获取，需要配置有效的 Token。

## 注意事项

1. 首次运行需要联网获取股票数据
2. 行业信息会缓存7天（`industry_cache.pkl`）
3. 调试模式会输出大量信息，建议只用于单只股票分析
4. 批量验证会生成较多日志文件，注意磁盘空间
